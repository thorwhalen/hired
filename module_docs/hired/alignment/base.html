hired.alignment.base

Schemas for JD-vs-candidate alignment analysis.

A job description is decomposed into Requirement records; each is analysed into a RequirementRecord carrying a three-valued MatchState, a Bucket, grounding Evidence, and the fields the elicitation loop needs. The whole analysis is an AlignmentReport — verdict-first, evidence-quoted, banded (never a single false-precision percentage).

See misc/docs/DESIGN.md §6.
