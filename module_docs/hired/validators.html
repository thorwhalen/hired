hired.validators

JSON Resume schema validation utilities.

Implement: - Schema loading and caching - Validation functions - Error reporting utilities

hired.validators.validate_and_normalize(raw: Mapping[str, Any], *, strict: bool = True) → ResumeContent[source]

Full pipeline validation: pydantic + json schema (optional strict).

Returns normalized ResumeContent. Raises ValidationError if pydantic validation fails or schema invalid in strict mode.

hired.validators.validate_resume_content(content: Mapping[str, Any], *, strict: bool = True) → bool[source]

Validate resume content against the JSON Resume schema.

If strict=False and schema fails, returns True (permissive mode).

hired.validators.validate_with_models(raw: Mapping[str, Any]) → ResumeContent[source]

Validate a raw resume dict using pydantic models first.

This normalizes fields like work.company/name before schema validation. Raises pydantic ValidationError if structure invalid.
